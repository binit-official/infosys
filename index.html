SECTION A ‚Äî THE TRUE THEORY OF ARRAYS

Arrays look simple, but 70% of Day‚Äë1 questions depend on understanding how arrays behave in memory.

1. Hardware Level Reality: An array is a single continuous block of memory. Accessing arr[i] is O(1) because the CPU computes address = base + (i √ó element_size) instantly.
2. Subarrays vs Subsets:
  • SUBARRAY: Contiguous. Order preserved. Example: [2, 4, 5] is a subarray of [1, 2, 4, 5, 9].
  • SUBSET: Not contiguous. Order doesn‚Äôt matter. Example: {1, 9, 5}.

Most Day-1 problems require thinking: ‚ÄúWhich contiguous part of the array affects this?‚Äù

SECTION B ‚Äî PREFIX SUM SYSTEM

Prefix sum is the MOST IMPORTANT concept today. It converts expensive subarray operations from O(N) to O(1).

The Formula: sum(l..r) = prefix[r] ‚Äì prefix[l-1]

Applications: Running Sum, Pivot Index, Range Sum Query, and Subarray Sum Equals K (Prefix + HashMap).

A. THE TWO-POINTER SYSTEM

Two pointers allow you to process pairs in a sorted array or shrink a search space without nested loops (O(N) instead of O(N¬≤)).

1. Opposite Ends (Left-Right): Used for sorted arrays (Two Sum II) or container problems. If sum/area is too small, logic dictates moving one specific pointer to improve it.

2. Slow-Fast (In-Place): Used for Move Zeroes or Remove Duplicates. Fast pointer scans for valid items, Slow pointer writes them to the "compacted" region.

3. Fixed + Scanning (3Sum/4Sum): Sort the array first. Fix one index i, then use Left/Right pointers on the remaining array to find pairs. This reduces O(N¬≥) to O(N¬≤).
B. SLIDING WINDOW SYSTEM

The window maintains a valid property (e.g., "sum < K" or "unique chars") while moving across the array.

Variable Size Window:
1. Expand Right to include new element.
2. Update counters (frequency map, sum, etc.).
3. While condition is broken (e.g., sum > K), shrink Left.
4. Update max/min length at every valid step.

Fixed Size Window: Used for Anagrams/Permutations. Window size is constant (e.g., Length of Pattern). You add one element at Right and immediately remove one element from Left.

C. MONOTONIC STACK SYSTEM

Used to find the "Next Greater" or "Next Smaller" element in O(N).

Logic: Maintain a stack of indices such that values are always decreasing (or increasing).
When a new element breaks this order (e.g., it is larger than the top), you POP from the stack. The new element is the "Next Greater Answer" for the popped element.

A. STRING FUNDAMENTALS

1. Characters ‚Üí Frequencies: Strings rarely need index values; they need relationships. We use int[26] (lowercase), int[128] (ASCII), or HashMaps. Infosys loves comparing two frequency arrays or counting items inside a window.

2. Sorting (Canonical Form): Anagrams share a signature. "eat", "tea", "ate" all sort to "aet". This allows grouping.

3. Palindrome Logic: Reads same forward and backward. Checked via Two Pointers (moving inward) or Expand Around Center (moving outward).

4. Sliding Window with Maps: The most powerful concept. Expand Right, Update Counts, Shrink Left while condition holds. Used in Minimum Window Substring.

B. DEEP THEORY OF PROBLEMS
Valid Anagram (242): Compare frequency arrays. If lengths differ, fail. If counts match (all zero difference), success.

Group Anagrams (49): Transform each string into a signature (Sorted String or Freq String "#1#0..."). Group by signature in a HashMap.

Valid Palindrome (125): Two phases: 1. Clean string (only alphanumeric, lowercase). 2. Two-pointer check from ends.

Longest Palindromic Substring (5): Expand from center. Odd length centers at i, Even length centers at i, i+1. Do NOT check all substrings (O(N¬≥)).

Palindromic Substrings (647): Same core method as LPS (Expand Around Center). Every index serves as Odd and Even center. Instead of tracking max, we increment COUNT for every match.

String Compression (443): Slow-Fast pointers. Read blocks of identical characters with Fast, write compressed version (Char + Count) with Slow.

Compare Version Numbers (165): Split by dot. Convert blocks to integers. Compare v1 > v2. If lengths differ, missing blocks are treated as 0.

Reorganize String (767): Greedy strategy using a Max Heap. Frequency count first. Always pick the top two most frequent characters to alternate them and avoid adjacency clashes. If any char freq > (N+1)/2, it is impossible.

Minimum Window Substring (76): The ultimate challenge.
1. Build `Need` map.
2. Expand `Right` until `Have == Required`.
3. Shrink `Left` to minimize window while it remains valid.
4. Track min length.

A. THE TRUE THEORY OF HASHMAPS & SETS

1. What is a HashMap? Stores data in Key -> Value pairs. Keys are unique. Internally uses hashing + buckets. Access is O(1) average. Use for fast frequency counts, pattern matching, and prefix tracking.

2. What is a HashSet? A HashMap without values. Use for uniqueness, fast O(1) lookups, and detecting duplicates.

Common Operations:
Map: put(k,v), get(k), containsKey(k), getOrDefault(k, def), remove(k).
Set: add(v), contains(v), remove(v), size().
B. MINDSET FOR HASHMAP PROBLEMS
  • Survival Rule: If checking "have we seen this before?" ‚Üí HashMap/Set.
  • Frequency Rule: If problem needs counts ‚Üí HashMap.
  • Sequence Rule: If checking for longest consecutive sequence ‚Üí HashSet.
  • Mapping Rule: If problem asks for relationships/patterns ‚Üí HashMap with indices/groups.
C. DEEP CONCEPTUAL BREAKDOWN

First Unique Character (387): Pure frequency problem. Phase 1: Count all. Phase 2: Scan string, find first with count 1. Cannot be done in 1 pass.

Word Pattern (290): Bijection problem. Must use TWO Maps (P->W and W->P) to ensure 1-to-1 mapping. Single map fails if different keys map to same value.

Intersection of Arrays (349): Set problem. Put Array 1 in Set. Iterate Array 2, check if present. Ensures uniqueness and O(1) lookup.

Longest Harmonious Subsequence (594): Frequency based. For every X, check if X+1 exists. Length = freq[x] + freq[x+1]. Order doesn't matter.

Longest Consecutive Sequence (128): Set-based O(N). Key insight: Only start counting sequence if x-1 is NOT in set (meaning x is the start).

Count Subarrays XOR = K: Prefix XOR + HashMap. Similar to "Sum = K". Prefix[R] ^ Prefix[L-1] = K means Prefix[L-1] = Prefix[R] ^ K. Count occurrences of required prefix.

Happy Number (202): Cycle detection. If sum of squares repeats a number seen before -> Infinite Loop -> Not Happy. Use HashSet to track seen numbers.

A. WHAT IS BACKTRACKING REALLY?

Backtracking is: ‚ÄúTry all possibilities smartly, and when one path is wrong or complete, go back and try another.‚Äù

Think of it like exploring all paths in a game: Choose -> Go Deeper -> Dead End? -> Undo (Backtrack) -> Try Different.

Every Backtracking Solution Has 3 Things:
1. State: What we have built so far (e.g., current subset, current string).
2. Choices: What we can choose next (e.g., include element, move up/down).
3. Constraints & Goal: When to stop (invalid choice) or accept result (base case).
B. DEEP CONCEPTUAL BREAKDOWN

Subsets (78): Pattern: "Each element: choose or not choose". State: current subset + index. No constraints, every path is valid.

Subsets II (90): Like Subsets, but array has duplicates. Trick: Sort array. If nums[i] == nums[i-1] and we skipped previous, skip current to avoid duplicate subsets.

Permutations (46): "Order matters, use every element once". Choices: Pick any unused element (use used[] array). Base case: length == n.

Permutations II (47): Duplicates exist. Trick: Sort array. Skip nums[i] if it equals nums[i-1] AND nums[i-1] was NOT used in this branch.

Combination Sum (39): Sum to target, reuse allowed. Choices: Pick current index again (stay at i) OR move to next. Constraint: Sum > Target.

Combination Sum II (40): No reuse, duplicates exist. Sort array. Pass i+1. Skip duplicates at same level.

Letter Combinations (17): Digits to Letters. Cartesian product. Depth = digits length. Branching = letters per digit.

Generate Parentheses (22): Constraint-based. Add '(' if open < n. Add ')' if close < open. Base: open == n && close == n.

Word Search (79): Grid DFS. Move 4 directions. Mark visited, recurse, unmark (backtrack). Constraints: Bounds, mismatch, already visited.

A. MENTAL MODEL OF TREES

1. What is a Binary Tree? It's a collection of nodes where each node has a value, a left child, and a right child. The top node is the Root. Since children are subtrees, we solve almost everything using Recursion.

2. DFS vs BFS (The Two Ways to Walk):
  • DFS (Depth-First Search): Go deep along one path before going sideways. Types: Preorder (Root-L-R), Inorder (L-Root-R), Postorder (L-R-Root).
  • BFS (Breadth-First Search): Go level by level (like waves). Uses a Queue. Each "round" processes one entire level.
B. DEEP CONCEPTUAL BREAKDOWN

1. Maximum Depth (104): The depth of a tree = 1 + max(depth of left, depth of right). Each node trusts its children to tell them their depth, then adds 1 for itself.

2. Inorder Traversal (94): Order: Left Subtree ‚Üí Root ‚Üí Right Subtree.
Why it matters: For a Binary Search Tree (BST), Inorder traversal gives values in sorted order.

3. Level Order Traversal (102): This is BFS. Start with Root in Queue. While Queue is not empty, process size() nodes (current level), and add their children (next level).

4. Right Side View (199): Imagine standing on the right. You see the last node of every level. We use BFS: The last node processed in the for(i < size) loop is the rightmost one.

5. Symmetric Tree (101): A tree is a mirror if: Root values match, AND Left.Left == Right.Right, AND Left.Right == Right.Left. Symmetry is about opposite sides.

6. Diameter of Binary Tree (543): The longest path between ANY two nodes. It might not pass through the root.
Logic: At every node, calculate LeftHeight + RightHeight. This is the longest path through that specific node. We track the global Max of this value.

7. Lowest Common Ancestor (236): "Where do paths to p and q diverge?"
Ask Left: "Do you have p or q?" Ask Right: "Do you have p or q?"
If BOTH say yes ‚Üí Current Root is the LCA.
If only one says yes ‚Üí Return that non-null answer up.

8. Path Sum (112): We are "paying off" the target sum as we go down. Subtract node value from Target. If we reach a Leaf and RemainingSum == 0, we found a path.

A. CORE IDEA: BFS ON NUMBERS / STATES

Most "minimum operations" problems are secretly Shortest Path in an Unweighted Graph.

  • Nodes: States (Numbers, Arrays, Strings, Configs).
  • Edges: Operations that transform one state to another.
  • Goal: Minimum edges from Start to Target.
KEY INSIGHT: If every operation has equal cost (1 step), BFS = Dijkstra for free. The first time you reach a state, it is via the shortest path.
B. DEEP CONCEPTUAL BREAKDOWN

1. Minimum Operations A ‚Üí B: Transform A to B using +1, -1, *2. Model as graph where nodes are integers. BFS from A to B.

A. WHAT IS 1D DP?

Dynamic Programming = Breaking a big problem into overlapping subproblems. 1D DP means your state depends on one index or number.
Shapes: Counting Ways, Minimization, Maximization, Feasibility (True/False).

Every DP needs:
1. State Definition: What does dp[i] mean?
2. Transition: How to compute dp[i] from smaller values?
3. Base Case: The smallest trivial answer (e.g., dp[0]).
B. DP MASTER PATTERNS (CRITICAL)
  • Pattern A (Fibonacci Style): Depends on last 1-2 states.
    Used in: Climbing Stairs, Min Cost Climb, Decode Ways.
  • Pattern B (Adjacent Restriction): Take or Skip.
    Used in: House Robber, House Robber II.
  • Pattern C (Unbounded Knapsack): Choose items infinite times.
    Used in: Coin Change (Min), Coin Change II (Count).
  • Pattern D (Partition/Boolean): Can we segment? Check all cut points.
    Used in: Word Break.
  • Pattern E (Max/Min Track): Negatives flip signs.
    Used in: Max Product Subarray.
C. DEEP CONCEPTUAL BREAKDOWN

Climbing Stairs (70): Count ways. Each step depends on previous 1 or 2. This is literally Fibonacci. dp[i] = dp[i-1] + dp[i-2].

Min Cost Climbing Stairs (746): Minimization. Pay cost to step. Reach top from n-1 or n-2. dp[i] = cost[i] + min(dp[i-1], dp[i-2]).

House Robber (198): Maximize sum, no adjacent. At each house: Take it (and skip previous) OR Skip it. dp[i] = max(dp[i-1], dp[i-2] + nums[i]).

House Robber II (213): Circular constraint. You cannot rob both First and Last. Trick: Run linear House Robber twice: once on 0..n-2 and once on 1..n-1. Take max.

Coin Change (322): Min coins (Unbounded Knapsack). Try every coin c. dp[x] = min(dp[x], dp[x-c] + 1). If impossible, return -1.

Coin Change II (518): Count combinations. Order doesn't matter ({1,2} same as {2,1}). Loop Coins OUTER, Amount INNER. dp[x] += dp[x-coin].

Decode Ways (91): Count decodings. Check single digit (1-9) -> add dp[i-1]. Check two digits (10-26) -> add dp[i-2]. '0' is tricky.

Word Break (139): Boolean Partition. Can s[0..i] be segmented? Try all cut points j. If dp[j] is true AND s[j..i] is in dict -> dp[i] = true.

Max Product Subarray (152): Negatives flip signs. Track BOTH Max and Min product at every step. If current is negative, swap max/min candidates.

A. BIG PICTURE: 2D / STRING DP

What is it? DP where state depends on two variables (e.g., index i and j). Used when comparing two sequences or moving in a grid.

Typical Interpretations:

B. DEEP CONCEPTUAL BREAKDOWN

1. Unique Paths (62): Grid counting. Reached from Top or Left. dp[i][j] = dp[i-1][j] + dp[i][j-1]. Combinatorics in disguise.

2. Unique Paths II (63): Obstacles block flow. If cell is obstacle ‚Üí dp[i][j] = 0. Otherwise sum neighbors.

3. Minimum Path Sum (64): Grid minimization. Cost to reach (i,j) = cell value + min(Top, Left). dp[i][j] = grid[i][j] + min(...).

4. LCS (1143): Compare two strings. If chars match ‚Üí 1 + dp[i-1][j-1] (diagonal). If not ‚Üí max(dp[i-1][j], dp[i][j-1]) (skip one).

8. Target Sum (494): Assign +/- signs. Insight: Convert to Subset Sum. Count subsets with sum P = (S + Total) / 2.

9. Distinct Subsequences (115): Count ways to form T from S. If chars match, two choices: Use it (move both) OR Skip it (move S only). dp[i][j] = dp[i-1][j] + (match ? dp[i-1][j-1] : 0).

A. WHAT IS GREEDY?

Definition: At every step, pick the locally best option and never undo it.
The Golden Question: "Why is this local choice always safe?"

The Exchange Argument (Proof):
If you can swap a non-greedy choice with a greedy one without hurting the result, then Greedy is optimal.
Example: If an optimal solution picks a large cookie for a small child, swapping it for a small cookie leaves the large one for a bigger child (better).
B. DEEP CONCEPTUAL BREAKDOWN

1. Assign Cookies (455): Feed max kids.
Greedy: Smallest valid cookie -> Smallest greed.
Why? Saving big cookies for high-greed children maximizes future matches.

2. Lemonade Change (860): Give correct change.
Greedy: For $20, always give $10+$5 if possible. Only use 3x$5 if forced.
Why? $5 bills are the "atomic unit" of change. Hoarding them is safer.

3. Jump Game (55): Reachability.

1. Single Number (136): "Every element twice except one". XOR cancels pairs (x^x=0). Only unique remains. Don't use map.

2. Missing Number (268): 0..N with one missing. XOR full range [0..N] vs Array. Pairs cancel. Missing number remains.

2. Sliding Window Template JAVA
public int slidingWindowGeneric(int[] arr, int K) {
    int n = arr.length, left = 0, right = 0, best = 0;
    Map<Integer, Integer> freq = new HashMap<>();

    while (right < n) {
        // Expand
        freq.put(arr[right], freq.getOrDefault(arr[right], 0) + 1);
        right++;

        // Shrink while invalid
        while (!isValidWindow(freq, K)) {
            freq.put(arr[left], freq.get(arr[left]) - 1);
            if (freq.get(arr[left]) == 0) freq.remove(arr[left]);
            left++;
        }
        best = Math.max(best, right - left);
    }
    return best;
}
private boolean isValidWindow(Map<Integer, Integer> freq, int K) {
    return freq.size() <= K; // Example: at most K distinct
}

public int stateBFS(int start, int target) {
    Queue<Integer> q = new ArrayDeque<>();
    Set<Integer> seen = new HashSet<>();
    q.offer(start); seen.add(start);
    int steps = 0;

    while(!q.isEmpty()) {
        int size = q.size();
        for(int i=0; i<size; i++) {
            int s = q.poll();
            if(s == target) return steps;
            for(int ns : neighbors(s)) {
                if(!seen.contains(ns)) {
                    seen.add(ns); q.offer(ns);
                }
            }
        }
        steps++;
    }
    return -1;
}
